## Ініціалізація та запуск

1. Клонувати репозиторій:

```bash
git clone https://github.com/Aivazov/bank-statement-analyzer.git
```

2. Встановити залежності:
```bash
npm install
```

3. Запустити сервер для розробки:
```bash
npm run dev
```

4. Відкрити [http://localhost:3000](http://localhost:3000) у своєму браузері, щоб переглянути результат.


## Труднощі

*   **Що було неочевидно:** Найбільше часу пішло на реалізацію надійної валідації CSV через Zod у поєднанні з потоковим парсингом PapaParse, щоб коректно відловлювати помилкові рядки та показувати їх кількість користувачу. Друге місце за складністю — statement.ts, де міститься логіка розрахунку топ-5 контрагентів за витратами
*   **Технічний вибір:** Використав паттерн «Чиста функція» для розрахунків у `lib/statement.ts`, що дозволило покрити бізнес-логіку тестами Vitest незалежно від UI.
*   **Особливості:** Додав бонусний функціонал експорту відфільтрованої таблиці назад у CSV та темну тему в стилі Zinc/Zinc-900 для комфортної роботи.

## Технологічний стек

*   **Framework:** Next.js 15 (App Router), React 19
*   **Language:** TypeScript (Strict mode)
*   **Styling:** Tailwind CSS + shadcn/ui
*   **Validation:** Zod
*   **CSV Parsing:** PapaParse
*   **Testing:** Vitest
